suspendAndResumeOnMain

suspend fun <R> suspendAndResumeOnMain(block: (MainContinuation<R>, InvokeOnCancellation) -> Unit): R
Content copied to clipboard

Suspends the current coroutine and calls block with a MainContinuation that will always dispatch on the main thread

block will initially be called in the main thread but it's ok to call MainContinuation.resumeWith from any thread, MainContinuation will ensure that the final result is delivered to the main thread. block should not throw. Errors are propagated by passing a Result to MainContinuation.resumeWith

Internally, this works by creating a StableRef to the CancellableContinuation so that it can be tunneled accross threads without being frozen. Because of this, this only works for coroutines that run in the main thread.
